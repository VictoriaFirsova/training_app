# Деплой бота на Railway

## 1. Подготовка

1. Зарегистрируйтесь на [railway.app](https://railway.app)
2. Убедитесь, что репозиторий проекта залит на GitHub (или GitLab)

## 2. Создание проекта

1. Нажмите **New Project**
2. Выберите **Deploy from GitHub repo**
3. Подключите GitHub и выберите репозиторий `training_app`
4. Railway автоматически определит Python и начнёт сборку

## 3. Добавление PostgreSQL (обязательно)

SQLite на Railway не сохраняет данные между перезапусками. Используйте PostgreSQL:

1. В проекте нажмите **+ New** → **Database** → **PostgreSQL**
2. Дождитесь создания БД
3. Railway автоматически добавит переменную `DATABASE_URL` в сервис бота
4. Нажмите на сервис бота → **Variables** → **Add Reference** → выберите `DATABASE_URL` из PostgreSQL

## 4. Переменные окружения

В сервисе бота откройте **Variables** и добавьте:

| Переменная    | Значение                         |
|---------------|----------------------------------|
| `BOT_TOKEN`   | Токен бота от @BotFather        |
| `TIMEZONE`    | `Asia/Tbilisi` (по умолчанию, для UTC+4) |
| `LOG_LEVEL`   | `DEBUG` для подробных логов, `INFO` (по умолчанию) |

`DATABASE_URL` добавляется автоматически при подключении PostgreSQL (шаг 3).

## 5. Проверка деплоя

1. Railway запустит бота после успешной сборки
2. В логах (**Deployments** → **View Logs**) должно появиться что-то вроде:
   ```
   Application started
   ```
3. Напишите боту в Telegram — он должен ответить

## 6. Обновление кода

```bash
git add .
git commit -m "обновление"
git push
```

Railway автоматически пересоберёт и перезапустит бота при каждом `git push`.

**Важно:** при каждом перезапуске (деплое) сбрасывается состояние диалога в памяти. Если пользователь был в середине тренировки или создания шаблона, он увидит сообщение: «Бот был перезапущен (обновление). Нажмите /start и начните заново.» Данные в БД (тренировки, упражнения) сохраняются, теряется только текущий контекст диалога.

## Структура БД

- **workout_templates** — шаблоны тренировок (создание через «1. Создать тренировку»)
- **template_exercises** — упражнения в шаблоне (добавление через «Редактировать» шаблона)
- **workout_sessions** — запущенные тренировки («4. Запустить тренировку»)
- **exercise_logs** — записи упражнений во время тренировки (подходы, повторения, вес)

Время хранится в UTC; при необходимости можно задать `TIMEZONE=Asia/Tbilisi`.

## Устранение проблем

- **Ошибка с faster-whisper** — пакет может не собраться на Railway из‑за размера. Если голосовой ввод не нужен в облаке, можно вынести его в optional dependency.
- **База пустая после деплоя** — это нормально, PostgreSQL создаётся заново. Локальные данные в облако не переносятся.
- **Бот не отвечает** — проверьте, что `BOT_TOKEN` указан и что в логах нет ошибок.
- **«Бот был перезапущен»** — после деплоя состояние диалога сбрасывается. Пользователю нужно нажать /start.
